> Turn workouts into currency. Thousands of concurrent users competing on leaderboards backed by fraud-resistant activity tracking.

## The Challenge

When fitness activity earns digital currency, people cheat. Overlapping activities from multiple tracking apps (Strava, Apple Health) can be submitted simultaneously to inflate scores. The system needed to handle thousands of concurrent telemetry signals while keeping leaderboards fair.

## Key Engineering Decisions

### Fraud-Resistant De-duplication

The `processActivities` pipeline detects time-overlapping entries from third-party fitness APIs. When conflicts arise, it retains the entry with the highest data fidelity and discards the rest.

```javascript
async function syncStravaActivity(userId, activity) {
  const overlap = await detectActivityOverlap(userId, activity.timestamp);

  if (overlap && overlap.fidelity > activity.fidelity) {
    return; // Discard low-fidelity overlapping attempt
  }

  gamificationEmitter.emit("activity_verified", { userId, activity });
}
```

### Event-Driven Gamification

Point calculations and leaderboard updates run asynchronously via custom **Emitters** and **Subscribers**, keeping the API response path fast and the main thread unblocked.

### Virtual Economy Ledger

A dual-entry "Coins" system using **MongoDB aggregations** — every transaction records both a ledger entry and a balance update, preventing double-spending with banking-grade integrity.

### Retrospective Challenge Backfill

When a user joins a challenge mid-way, the system automatically scans their past activities and backfills eligible entries. No missed points for late joiners.

## Impact

- **Zero fraudulent payouts** — Overlap detection catches duplicate submissions across all connected fitness APIs
- **Real-time leaderboards** — Async event processing keeps rankings updated without blocking API responses
- **Fair late-join experience** — Retrospective backfill ensures competitive fairness regardless of join date
