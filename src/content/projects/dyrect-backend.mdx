> Scan a QR code, register a product, activate a warranty — at scale across international brands.

## The Challenge

Physical QR codes on consumer products generate extremely bursty, unpredictable traffic. A single ad campaign can spike scan volume 100x overnight. With multiple international brands sharing the infrastructure, a standard MongoDB collection with tens of millions of short-URLs quickly hits index degradation.

## Key Engineering Decisions

### Dynamic Collection Sharding

Instead of one massive collection, telemetry routes into brand-specific MongoDB collections on a shared-nothing basis. Each brand's scan data lives in its own partition, eliminating B-Tree bloat.

```javascript
async function logProductScan(domainHost, telemetryPayload) {
  const collectionName = `scans_${domainHost.replace(/\./g, "_")}`;
  const PartitionedCollection = mongoose.model(collectionName, TelemetrySchema, collectionName);

  await PartitionedCollection.create(telemetryPayload);
}
```

### Traffic Spike Absorption

**RabbitMQ** queues buffer incoming scan events during campaign spikes, ensuring the database never receives more writes than it can handle.

### Secure Enterprise Access

For tenants requiring direct database access, isolated **SSH tunnels** connect to private VPC resources — no multi-tenant API endpoints exposed.

## Impact

- **Handles 100x traffic spikes** — RabbitMQ buffering absorbs campaign-driven scan bursts
- **Constant-time URL resolution** — Per-brand sharding keeps index sizes small regardless of total platform scale
- **Secure direct access** — Enterprise tenants connect via isolated VPN tunnels, not shared APIs
