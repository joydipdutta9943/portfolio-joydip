> Construction project management with hierarchical permissions, optimized GraphQL, and native Microsoft Outlook sync.

## The Challenge

Construction projects have deeply nested hierarchies: companies → portfolios → sites → sub-contractors. Every API query must respect multi-tiered permissions, and recurring project meetings need to sync seamlessly with Microsoft Outlook — not as a separate calendar, but inside the user's existing workflow.

## Key Engineering Decisions

### GraphQL AST Optimization

Instead of `SELECT *`, resolvers parse the incoming GraphQL AST to extract only the requested fields. These map directly to Prisma `select` clauses, cutting network payload and query time by roughly 60% across the API surface.

### Hierarchical RBAC

Middleware evaluates each request's scope — company-level or project-level — and enforces boundaries on every GraphQL resolver. No resolver runs without a permission check.

### Microsoft Graph Calendar Sync

Meeting recurrence patterns sync bi-directionally with **Microsoft Graph API**, placing project meetings directly in users' existing Outlook calendars rather than maintaining a separate internal calendar.

## Impact

- **~60% faster queries** — AST-based field selection eliminates over-fetching across all resolvers
- **Native Outlook integration** — Meetings appear in users' existing calendars with full recurrence support
- **Strict hierarchy enforcement** — Every resolver scoped to the correct company/project permission boundary
