> An enterprise-grade, multi-tenant wealth management platform orchestrating investment accounts, KYC verification, and AI-driven recommendations via a RESTful API serving 47+ controllers across 5 business domains.

## Key Architecture & Engineering

### Two-Layer Access Control (ACL)

Combines API-level middleware with a strict row-level security system across all 61 data models. It uses six role-specific handlers and **Redis caching** to accurately compute accessible records for different users.

### Post-Commit Callback Pattern

Ensures that external operations (like sending emails or triggering _WealthKernel_ API calls) only execute after the database transaction **safely commits**, preventing inconsistent data states if a rollback occurs.

### Event-Driven Webhook Pipeline

Manages 31 different _WealthKernel_ webhook event types routed through **BullMQ** queues. It uses `HMAC-SHA256` signature verification and idempotency guards to prevent duplicate transaction processing.

```typescript
// Conceptual BullMQ Webhook Pipeline
const webhookQueue = new Queue("wealthkernel_events", { connection: redis });

webhookQueue.process(async (job) => {
  // 1. Verify HMAC-SHA256 Signature
  verifySignature(job.data.payload, job.data.signature);

  // 2. Check Idempotency Key in Redis
  if (await redis.exists(`processed:${job.data.eventId}`)) return;

  // 3. Post-Commit Callback Execution
  await db.transaction(async (tx) => {
    await routeEventLogic(tx, job.data.payload);
  });

  await redis.set(`processed:${job.data.eventId}`, "true");
});
```

### Field-Level Encryption

Uses `AES-256-GCM` encryption with zero-downtime key rotation to protect sensitive Personally Identifiable Information (PII), achieving **ISO 27001 compliance** while maintaining extremely fast query latency.
