> A digital agriculture ecosystem physically validating digital contract signatures against dynamic geospatial boundaries.

## Key Architecture & Engineering

### Real-Time Negotiation Engine

Built a transactional event engine that aggressively handles price and quantity bidding. It natively validates proposed crop quantities against the farmer's physical geospatial land availability (`area_in_use`) and locks resources dynamically upon acceptance, ensuring **100% conflict-free contracts**.

### Automated Agricultural Scheduler

Once a contract is mathematically verified and signed, a recurring task engine automatically clones a master blueprint into a personalized, day-by-day crop care schedule (sowing, watering, fertilizing) for the farmer's specific topology constraint.

### Unified Multi-Tenant API

Designed a singular Express server cluster that exposes distinct, logically separated **GraphQL schemas** for three different user personas (`api/admin`, `api/business_app`, `api/farmer_app`) to maximize rigorous security and maintainability within a highly streamlined monolithic repository.
