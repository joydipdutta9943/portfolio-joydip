## The Challenge

The school needed two things from one website: a lightning-fast public portal for parents to check announcements and notices, and a robust interface for managing structured CBSE compliance data (enrollments, financial audits, staff metrics). The previous site was a bloated monolith that failed modern Lighthouse benchmarks.

## Key Engineering Decisions

### Islands Architecture

**Astro v5** partial hydration (`client:visible` / `client:idle`) strips unused JavaScript from the main thread. Static content ships as pure HTML; React components only hydrate when needed.

```html
<Layout title="AGNIBINA MODEL SCHOOL">
  <!-- Zero-JS static rendering for the hero -->
  <HeroSection announcements="{latestNotices}" />

  <!-- React island: hydrates only when scrolled into view -->
  <ComplexComplianceDataGrid client:visible cbseData="{fetchRegulatoryStats()}" />
</Layout>
```

### Design System

A custom **Tailwind CSS v4** design system ensuring responsive layouts and accessibility compliance across all screen sizes.

### Automated CBSE Compliance Module

A React pipeline that ingests raw statistical data (enrollments, financial audits, staff metrics) and formats it into the specific tables and structures required by government auditors. Replaced weeks of manual data entry.

## Impact

- **100/100 Lighthouse** — Perfect scores across Performance, Accessibility, Best Practices, and SEO
- **Near-instant load times** — SSG + partial hydration delivers sub-second Time to Interactive
- **Automated compliance** — CBSE disclosure module eliminated weeks of manual formatting per audit cycle
