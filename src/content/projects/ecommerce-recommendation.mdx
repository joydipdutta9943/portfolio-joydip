> Personalized product recommendations in under 15ms — with a clean fallback for first-time visitors.

## The Challenge

Real-time recommendations during browsing are computationally expensive. Calculating collaborative similarity across thousands of products and growing user histories is an O(N²) operation that can't run inside a synchronous HTTP request. New users with no history face the "cold start" problem — an empty feed on their first visit.

## Key Engineering Decisions

### Async Matrix Computation

Heavy recommendation calculations run on distributed **Celery** workers. They process event streams (clicks, cart-adds) from PostgreSQL, compute cosine similarity via `scikit-learn`, and serialize results into **Redis**. The FastAPI layer only reads pre-computed scores — the expensive O(N²) math never runs inside a user-facing request.

### Cold Start Fallback

- **Returning users** — Personalized collaborative filtering (item-item / user-user similarity) computed hourly in the background
- **New users** — Instantly served from a cached "Trending" feed based on global momentum, ensuring no empty screens

### Pre-Computed Serving

Celery workers refresh recommendation scores into Redis every hour. The **FastAPI** layer operates exclusively on key-value lookups — no computation at request time.

## Impact

- **< 15ms response time** — All recommendations served from pre-computed Redis cache
- **Zero empty feeds** — Cold start fallback ensures every visitor sees relevant products from their first page view
- **Hourly freshness** — Background workers keep recommendations current without impacting API latency
