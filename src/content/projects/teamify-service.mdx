> Architecting complex financial ledgering, multi-tiered payout pipelines, and a timezone-aware recurrence engine.

## The Challenge

Building an isolated B2B marketplace requires orchestrating intense logic between booking states, vendor availability, and corporate credit authorizations. Hand-rolling cron jobs to manage thousands of concurrent Stripe pre-authorizations frequently results in stale locks and failed payments.

## The Solution

I designed the core marketplace backend.

### 1. Complex Financial Ledgering

I integrated **Stripe Connect** to manage the highly complex split payments between the system platform fee, the primary vendor, and local tax requirements. I structured robust logic to execute partial or full refunds accurately, even when the vendor had already drawn down their Stripe bucket balance.

```typescript
// Conceptual Multi-Tier Payment Split Workflow
async function captureCorporateBooking(bookingId) {
  const transaction = await mongoDB.Bookings.findById(bookingId);
  const provider = await mongoDB.Vendors.findById(transaction.vendorId);

  // Platform fee logic defined mathematically on ledger bounds
  const applicationFee = calculatePlatformPercentage(transaction.amount, provider.tier);

  const paymentIntent = await stripe.paymentIntents.capture(transaction.preAuthId, {
    amount_to_capture: transaction.amount,
    application_fee_amount: applicationFee,
    transfer_data: {
      destination: provider.stripeAccountId,
    },
  });

  return paymentIntent;
}
```

### 2. Timezone-Aware Recurrence Scheduling

Vendors dictate their availability across various geopolitical zones. I engineered an automated, dynamic event generator that factors in exact daylight savings variables and global UTC offsets when constructing 60-day recurring vendor availability slots natively in MongoDB.

### 3. Automated Lifecycle

Instead of cron loops that die on V8 limits, I utilized **Hatchet** background workflows.

- Automatically cancel highly stale, pending corporate bookings.
- Void pending Stripe Connect pre-authorizations seamlessly without requiring human administrator interventions.

## The Impact

- **Frictionless Money Flow:** Executed complex Split-Pay topologies safely across international vendors.
- **Workflow State Machines:** Eliminating generic cron loops for Hatchet's state machines resulted in zero dropped state mutations during booking expirations.
