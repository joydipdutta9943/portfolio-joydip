> Building a deterministic search experience over highly non-deterministic, multi-modal enterprise data.

## The Architecture of Ambiguity

Enterprise businesses are drowning in Dark Data—unstructured PDFs, embedded charts, and scanned documents that traditional indexing simply cannot parse. Relying on keyword matching (`Elasticsearch` or `Solr`) falls apart when users ask semantic, context-heavy questions.

The goal was to build a highly accurate **Retrieval-Augmented Generation (RAG)** pipeline that didn't just hallucinate answers, but could actually "read" and cite complex multi-modal documents.

```typescript
// Core Vector Ingestion Pipeline Concept
async function ingestMultiModalDocument(asset: FileBuffer) {
  // 1. OCR & Layout analysis via Gemini Vision
  const structuredText = await geminiVision.extractLayout(asset);

  // 2. Semantic Chunking (preserving paragraph boundaries)
  const chunks = await textSplitter.split(structuredText, { overlap: 200 });

  // 3. 768-Dimensional Embedding Generation
  const embeddings = await embedder.generate(chunks);

  // 4. Persistence in Atlas Vector Search
  await mongoVectorStore.insertMany(embeddings, { enforceSchema: true });
}
```

## The Execution

I designed a robust orchestration layer using **LangChain** in TypeScript. When a user queries the system, the pipeline operates in three distinct phases:

1. **Query Reformulation:** The raw user input is processed by an LLM to generate a dense, semantic search vector.
2. **K-Nearest Neighbor (k-NN) Retrieval:** `MongoDB Atlas Vector Search` executes a high-speed cosine similarity search across millions of 768-dimensional document embeddings.
3. **Multi-Modal Synthesis:** The retrieved chunks—alongside raw image buffers if necessary—are injected into the **Google Gemini Pro** context window to synthesize a definitive, cited response.

### Taming Latency

Vector math is computationally expensive. To prevent the API from timing out during heavy enterprise usage, I implemented an intelligent `Redis` caching layer mapping exact semantic query hashes to pre-generated responses, effectively bypassing the pipeline for frequently asked data.

## The Impact

- **85% Accuracy Target:** Achieved sustained factual accuracy across multi-modal queries for over 1,000 active users, practically eliminating hallucinations via strict context-bounding.
- **Microsecond Retrieval:** Reduced P99 API latency by **45%** utilizing Redis caching alongside Atlas Vector indexes.
- **Enterprise Security:** Built a strict **"Bring Your Own Key" (BYOK)** tenant model, cryptographically securing client API keys ensuring no cross-contamination of proprietary LLM environments.
