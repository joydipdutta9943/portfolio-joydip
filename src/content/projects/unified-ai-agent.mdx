> Ask a question about any PDF, image, or document — get an accurate, sourced answer in seconds.

## The Challenge

Enterprise users needed to query large collections of unstructured data — scanned PDFs, charts, images with embedded text — using natural language. A simple keyword search won't work when the answer lives inside a pie chart on page 47 of a PDF.

## Key Engineering Decisions

### LangChain Orchestration

LangChain acts as the routing layer, dynamically selecting the right tool (vector search, vision analysis, or summarization) based on the user's query intent. No hardcoded if/else chains.

### Multi-Modal Ingestion

PDFs parsed via `pdf-parse`, images processed through `sharp` and **Tesseract OCR**, complex visuals interpreted by **Gemini Vision**. Everything gets converted to 768-dimensional embeddings via Google's `text-embedding-004` before landing in the vector store.

### Hybrid Vector Search

**MongoDB Atlas Vector Search** combines semantic cosine similarity with keyword filtering. **Redis** caches frequent queries to avoid redundant embedding computations.

### Resilient Frontend

Custom API client with a **40-second timeout** for long-running LLM inferences. Optimistic UI updates display user messages instantly while the model processes in the background.

## Impact

- **Multi-format support** — PDFs, images, charts, and raw text all queryable through a single interface
- **Sub-second cached retrieval** — Redis caching on frequent queries eliminates redundant LLM calls
- **Seamless UX** — Optimistic updates mask inference latency, keeping the interface responsive
